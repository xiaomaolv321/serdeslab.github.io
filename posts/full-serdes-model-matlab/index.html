Introduction#

System-level SerDes modeling is frequently discussed, but complete and reproducible examples are surprisingly rare. Many discussions focus on individual blocks â€” such as TX FFE, CTLE, or CDR â€” without showing how these pieces interact in a full link simulation.

In this article, I demonstrate how to quickly assemble a complete SerDes behavioral model in MATLAB, suitable for architectural exploration, algorithm development, and design trade-off analysis.

The emphasis is on clarity, reproducibility, and engineering intuition, rather than transistor-level accuracy.

Overall Architecture#

The system model follows a typical high-speed serial link structure:

  • Transmitter (TX)

    • Binary data source
    • Feed-Forward Equalizer (FFE)

  • Channel

    • Frequency-dependent loss
    • ISI generation
    • Optional S-parameter based modeling

  • Receiver (RX)

    • Continuous-Time Linear Equalizer (CTLE)
    • Decision Feedback Equalizer (DFE)

  • Clock and Data Recovery (CDR)

    • Bang-Bang phase detector
    • Loop filter
    • Phase accumulator

All blocks are modeled behaviorally and can be simulated efficiently over long sequences.

Why System-Level Modeling Matters#

A full-link behavioral model allows you to:

  • Understand interactions between blocks
  • Evaluate ISI, jitter, and equalization trade-offs
  • Prototype CDR algorithms quickly
  • Debug issues that are invisible in isolated block simulations

Most importantly, it provides a common language between architecture, algorithm, and circuit design teams.

Transmitter Modeling: FFE as Pulse Shaping#

The TX FFE is modeled as a discrete-time FIR filter operating on the symbol stream. Instead of treating FFE as a black box, this approach highlights its role as pulse shaping before the channel.

Key points:

  • Pre-cursor and post-cursor taps directly control ISI distribution
  • Tap values can be expressed in linear scale or dB
  • The output waveform is generated at an oversampled rate

This representation makes it easy to explore trade-offs between TX and RX equalization.

Channel Effects and ISI Generation#

The channel is the primary source of inter-symbol interference.

Depending on the desired fidelity, the channel can be modeled as:

  • An impulse response derived from frequency loss
  • A sampled pulse response
  • An S-parameter based frequency-domain model

In all cases, the goal is to accurately capture post-cursor ISI, which dominates high-speed link performance.

Receiver Equalization#

On the RX side, equalization is split into two conceptual stages:

CTLE#

  • Compensates for high-frequency channel loss
  • Shapes noise and jitter transfer characteristics
  • Modeled as a continuous-time filter applied to the waveform

DFE#

  • Cancels post-cursor ISI using past decisions
  • Highly effective, but sensitive to error propagation
  • Modeled with discrete-time feedback taps

Separating these two stages clarifies their distinct roles in the link.

Bang-Bang CDR Modeling#

The CDR is implemented using a bang-bang phase detector, which makes binary early/late decisions based on data transitions.

The loop consists of:

  • Phase detector
  • Loop filter
  • Phase accumulator
  • Sampling phase control

This structure allows direct observation of:

  • Phase convergence behavior
  • Limit cycles
  • Sensitivity to jitter and ISI

Because the entire loop is behavioral, long-term stability and rare events can be simulated efficiently.

Simulation Results and Observations#

With this system-level model, you can easily generate:

  • Eye diagrams at different points in the link
  • CDR phase convergence plots
  • BER trends versus equalization settings

More importantly, you can observe cause-and-effect relationships rather than isolated metrics.

Reproducibility and Extensions#

All components in this model are algorithmic and reproducible. No proprietary PDKs or transistor models are required.

Possible extensions include:

  • Adaptive FFE / DFE algorithms
  • Jitter injection and tolerance analysis
  • Multi-rate or PAM4 signaling
  • Behavioral models calibrated against silicon
